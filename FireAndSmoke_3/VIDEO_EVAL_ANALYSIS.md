# 视频测试结果分析报告

测试目录：`D:\Researching\Yolo\Yolo\docs\video`

本报告用于解释前两轮模型在本地视频测试中的实际表现，并为第三轮训练提供依据。测试重点不是只看验证集 mAP，而是观察真实视频里的多火点、小火源、弱烟、灯光干扰和检测延迟。

## 1. 对比模型

| 模型 | 说明 |
|---|---|
| `sensors_yolov8m_2cls_100` | 第一轮 Sensors 2类模型 |
| `sensors_yolov8m_3cls_100` | 第一轮 Sensors 3类模型 |
| `aug_yolov8m_2cls_150` | 第二轮 Sensors + 自有数据 2类模型 |
| `aug_yolov8m_3cls_150` | 第二轮 Sensors + 自有数据 3类模型 |

## 2. 核心结论

第一轮 Sensors 模型在外部视频中的整体表现比第二轮融合模型更稳定，尤其是 `sensors_yolov8m_3cls_100` 更适合作为下一阶段的主线基线。

第二轮模型虽然加入了自有数据并训练 150 epoch，但没有明显提升多目标召回。主要原因不是训练轮次不够，而是数据源标注风格和类别结构不一致。

## 3. 主要问题

1. 多个小目标同时出现时，模型容易只检测到最明显的火焰或烟雾。
2. 远景火点在 `imgsz=640` 下被压缩得太小，特征被下采样削弱。
3. 早期烟雾低对比、边界模糊，单帧 YOLO 很难稳定识别。
4. 第二轮加入自有 fire_pic 后，fire/smoke 样本增加，但没有同步增加 other hard negative，3类模型的干扰物判断边界没有加强。
5. 不同数据源对火焰和烟雾的框选习惯可能不同，有的标整片火焰，有的只标局部火苗，这会影响模型学习到的输出风格。

## 4. 数据审计后的补充结论

第三轮审计发现：

- Sensors 3类数据：27294 张图，fire 26717，smoke 24560，other 14671，tiny 715。
- 自有 fire_pic：16039 张图，fire 29137，smoke 12958，tiny 4121。

这说明自有数据的小目标样本价值很高，但它没有 other 类。如果直接混入 3类训练，会增加 fire/smoke 占比，却不增加 other 负样本，容易削弱误报抑制。

## 5. 第三轮建议

优先训练 `YOLOv8m-P2 Sensors 3类`：

- 使用第一轮 Sensors 3类 best.pt 作为初始权重；
- 使用 P2 小目标检测头；
- 训练尺寸从 960 起步；
- 先不混入自有数据；
- 用 tiny/small/medium/large 分桶召回率评估小目标提升。

然后训练 `YOLOv8m-P2 Sensors 2类` 作为对照，判断 other 类是否值得保留。

后续如果要使用自有 fire_pic，建议先作为 2类小目标增强实验，或者在补齐 other hard negative 后再进入 3类训练。
