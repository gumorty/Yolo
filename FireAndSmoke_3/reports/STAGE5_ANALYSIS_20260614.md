## 三、失败原因分析

### 3.1 为什么 Stage5 没有提升？

| 因素 | 分析 |
|------|------|
| **起始点过高** | V4 best 本身已在良好局部最优，lr0=0.001 的微调难以跳出 |
| **增强过强** | mixup=0.05 + copy_paste=0.1 + erasing=0.2 增加训练难度，小目标检测受益有限 |
| **数据能力上限** | 75,744 张训练图片中，小目标标注质量可能是瓶颈而非模型能力 |
| **架构上限** | YOLOv8m-p2 在 960px 分辨率下的感受野和特征表达可能已达极限 |

### 3.2 V4 vs Stage5 训练模式对比

| 特征 | V4 (从头训) | Stage5 (fine-tune) |
|------|------------|-------------------|
| 起始 mAP50-95 | 0.553 (epoch 1) | 0.579 (epoch 1) |
| 峰值 mAP50-95 | **0.584 (epoch 16)** | 0.579 (epoch 12) |
| 提升幅度 | +0.031 | +0.0003 |
| 过拟合起点 | ~epoch 16 | ~epoch 1 (几乎立即) |
| 最佳 Val box loss | 0.990 | 0.999 |

**核心洞察**：V4 从头训练有充分的探索空间（0.553→0.584），而 Stage5 从 0.579 起步后几乎无法再提升，说明 V4 best 已非常接近该数据/架构组合的全局最优。
