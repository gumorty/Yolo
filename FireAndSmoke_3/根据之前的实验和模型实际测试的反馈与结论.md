# 根据已有实验和模型实际测试得到的反馈与结论

## 1. 当前问题本质

当前项目已经完成了两轮训练：

1. 第一轮：复现 Sensors FireAndSmoke 数据集，训练 YOLOv8m 2类、YOLOv8m 3类、YOLOv10m 3类。
2. 第二轮：把 Sensors 数据和自有 fire_pic 数据合并，继续训练 YOLOv8m 2类、YOLOv8m 3类。

第二轮训练数据更多、epoch 更多，但外部视频测试并没有表现出预期提升。这说明当前瓶颈不是简单的训练轮次，而是：

- 数据源之间标注风格不一致；
- other hard negative 没有同步增强；
- 大场景小目标在 640 输入下损失严重；
- 单帧 YOLO 对早期弱烟天然不敏感；
- 缺少真实红外热成像数据，无法完成真正的 RGB-T 融合预警。

## 2. 为什么不能继续盲目混合数据

第二轮融合后的 2类模型 Precision 略升，但 Recall 和 mAP 没有超过第一轮 Sensors 2类。3类模型也没有超过第一轮 Sensors 3类。

这说明混入自有数据后，模型可能更保守，但没有变得“检测更全”。原因是自有数据虽然增加了 fire/smoke，尤其增加了很多 tiny 小目标，但没有提供 other 类。如果继续直接混合，3类模型会学习到不平衡的类别边界。

## 3. 第三轮应该优先解决什么

第三轮重点不是换一个更大的模型，而是解决无人机巡检下的关键问题：

1. 小目标：增加 P2 检测头，保留更高分辨率特征。
2. 大场景：训练后配合 full 960/1280 和 sliced inference。
3. 多目标：提高 NMS IoU、切片检测和小目标分桶评估。
4. 误报：保留 3类模型，继续利用 other hard negative。
5. 早期预警：真实红外热成像数据进入后，做 RGB-T 融合，而不是用手填传感器数值伪装多模态。

## 4. 推荐主线

第三轮主线模型：

- `YOLOv8m-P2 Sensors 3类`
- 初始权重：第一轮 `sensors_yolov8m_3cls_100_best.pt`
- 输入尺寸：先 `imgsz=960`
- 类别：fire / other / smoke

对照模型：

- `YOLOv8m-P2 Sensors 2类`
- 初始权重：第一轮 `sensors_yolov8m_2cls_100_best.pt`
- 类别：fire / smoke

这样设计可以回答两个关键问题：

1. P2 检测头是否提升远处小火苗、小烟雾召回？
2. 保留 other 类是否比 2类模型更适合真实部署？

## 5. 红外热成像方向

红外热成像不是普通表格传感器。它是空间图像模态，能提供每个区域的热响应，因此可以和 RGB 检测框进行空间对齐。

下一步如果拿到 RGB-T 数据，应先做配对审计：

- RGB 和 Thermal 是否一一对应；
- 分辨率是否一致；
- 标签是否能映射到两个模态；
- 是否包含夜间、浓烟、远景、小火源场景。

只有通过审计，才进入 RGB-T 双流 YOLO 或后期融合实验。

## 6. 论文实验价值

第三轮工程形成的论文主线可以是：

面向无人机巡检的火焰烟雾小目标检测与 RGB-T 早期火灾预警方法。

可验证创新点：

- P2 小目标检测头提升远景小火源召回；
- full + sliced 推理改善大场景多火点漏检；
- other hard negative 抑制灯光/反光误报；
- RGB-T 数据接入后利用热红外增强早期火源判断。
